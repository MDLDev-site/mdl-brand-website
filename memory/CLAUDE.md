# memory/

Layered memory system for cross-session team knowledge.

## Structure

| Directory | Layer | Purpose |
|-----------|-------|---------|
| `MEMORY.md` | Index | Master index — always loaded, points to everything |
| `projects/` | L1 | Per-project context files — loaded on demand |
| `claims/` | L2 | Self-documenting team lessons — filenames are the lessons |
| `sessions/` | L2 | Optional session logs in `YYYY-MM-DD.md` format |

## Loading Protocol

See `MEMORY.md` for the complete loading rules. Key principle: **never load all project files at once** — load only what's relevant to the current task.

## Creating Claims

Claims are lessons learned. The filename IS the lesson title:

```
memory/claims/pool-pdas-must-be-excluded-from-concentration.md
```

Include inside: context, date learned, related claims.

## Cross-System Integration

This file-based memory works alongside:
- **OpenMemory MCP** — cross-project team patterns
- **GitNexus MCP** — code-level intelligence
- **Context7 MCP** — library documentation
